# NST 期权协议 - 保证金与结算计算公式

## 概述

本文档详细说明 NST 期权协议中的核心计算公式，包括建仓费用、保证金计算、价格变动时的保证金调整，以及订单结束时的盈亏结算。

---

## 1. 建仓费用计算

### 1.1 买方建仓费用

| 费用项 | 公式 | 说明 |
|--------|------|------|
| 建仓手续费 | `creationFee = 1 USDT` | 固定费用，不退还 |
| 期权费 | `premiumAmount = notionalUSDT × premiumRate / 10000` | 成交时支付给卖方 |
| 交易手续费 | `tradingFee = notionalUSDT × tradingFeeRate / 10000` | 0.1% (10基点) |

**买方总支付 = 建仓手续费 + 期权费 + 交易手续费**

```
示例：
- 名义本金: 100,000 USDT
- 费率: 6.5% (650 基点)

期权费 = 100,000 × 650 / 10000 = 6,500 USDT
交易手续费 = 100,000 × 10 / 10000 = 100 USDT
总支付 = 1 + 6,500 + 100 = 6,601 USDT
```

### 1.2 卖方建仓费用

| 费用项 | 公式 | 说明 |
|--------|------|------|
| 建仓手续费 | `creationFee = 1 USDT` | 固定费用 |
| 初始保证金 | `initialMargin = notionalUSDT × marginRate / 10000` | 报价时锁定 |

## 2. 保证金计算

### 2.1 初始保证金

```solidity
initialMargin = notionalUSDT × marginRate / 10000
```

| 卖方类型 | 最低保证金率 | 说明 |
|----------|-------------|------|
| 自由卖方 | ≥ 10% (1000 基点) | 由 Config.minMarginRate 配置 |
| 席位卖方 | 0% (可选) | 由 Config.seatSellerInitialMargin 配置 |

```
示例：
- 名义本金: 100,000 USDT
- 保证金率: 15% (1500 基点)

初始保证金 = 100,000 × 1500 / 10000 = 15,000 USDT
```

### 2.2 保证金调整

#### 追加保证金 (Add Margin)
```solidity
currentMargin = currentMargin + addedAmount
```
- 卖方可随时追加保证金
- 无上限限制

#### 提取超额保证金 (Withdraw Excess)
```solidity
excessMargin = currentMargin - initialMargin
withdrawableAmount = min(requestedAmount, excessMargin)
currentMargin = currentMargin - withdrawableAmount
```
- **限制**: 不能低于初始保证金

## 3. 价格变动与强平规则

### 3.1 强平触发条件

根据 `LiquidationRule` 配置，有两种触发模式：

#### 模式 A: 连续天数触发 (ConsecutiveDays)
```
if (连续上涨/下跌天数 >= consecutiveDays) && (单日涨跌幅 >= dailyLimitPercent) {
    触发强平
}
```

#### 模式 B: 单日大幅波动触发
```
if (单日涨跌幅 >= dailyLimitPercent × 2) {
    触发强平
}
```

### 3.2 追保计算

当价格变动导致盈亏接近保证金时：

```
requiredMargin = max(initialMargin, 估算亏损 × 安全系数)

if currentMargin < requiredMargin {
    需要追加 = requiredMargin - currentMargin
    追保截止时间 = isCrypto ? 2小时 : 12小时
}
```

## 4. 结算计算公式

### 4.1 盈亏计算 (Settle)

#### 看涨期权 (Call)

```solidity
if (finalPrice > strikePrice) {
    buyerProfit = (finalPrice - strikePrice) × notionalUSDT / strikePrice
} else {
    buyerProfit = 0
}
```

#### 看跌期权 (Put)

```solidity
if (strikePrice > finalPrice) {
    buyerProfit = (strikePrice - finalPrice) × notionalUSDT / strikePrice
} else {
    buyerProfit = 0
}
```

### 4.2 资金分配

```solidity
// 买方盈利受保证金限制
actualBuyerProfit = min(buyerProfit, currentMargin)

// 买方获得
buyerPayout = actualBuyerProfit

// 卖方获得 (剩余保证金)
sellerPayout = currentMargin - actualBuyerProfit
```

### 4.3 完整结算示例

```
场景：看涨期权 (Call)
- 名义本金: 100,000 USDT
- 开仓价 (strikePrice): $50,000
- 最终价 (finalPrice): $55,000
- 卖方保证金: 15,000 USDT

计算:
1. 价格涨幅 = (55,000 - 50,000) / 50,000 = 10%
2. 买方理论盈利 = (55,000 - 50,000) × 100,000 / 50,000 = 10,000 USDT
3. 保证金足够覆盖:
   - 买方获得: 10,000 USDT
   - 卖方获得: 15,000 - 10,000 = 5,000 USDT

如果最终价是 $70,000 (40% 涨幅):
1. 买方理论盈利 = (70,000 - 50,000) × 100,000 / 50,000 = 40,000 USDT
2. 超过保证金，按保证金限额:
   - 买方获得: 15,000 USDT (保证金上限)
   - 卖方获得: 0 USDT
```

## 7. 代码参考

核心结算逻辑位于: `contracts/core/OptionsCore.sol` 第 448-513 行

```solidity
// 看涨期权盈亏计算
if (order.direction == Direction.Call) {
    if (finalPrice > strikePrice) {
        buyerProfit = (finalPrice - strikePrice) * order.notionalUSDT / strikePrice;
    }
} else {
    // 看跌期权盈亏计算
    if (strikePrice > finalPrice) {
        buyerProfit = (strikePrice - finalPrice) * order.notionalUSDT / strikePrice;
    }
}

// 限制盈利不超过卖方保证金
if (buyerProfit > order.currentMargin) {
    buyerProfit = order.currentMargin;
}
```
