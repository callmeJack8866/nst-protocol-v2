## 4️⃣ 创建 RFQ 测试 (核心流程)

### 前置条件

- 已连接钱包
- 钱包有足够 USDT 余额

### 步骤 1: 证券档案

1. 点击 **"发起 RFQ"** 按钮进入创建页面
2. 填写资产信息：
   - **资产名称**: 贵州茅台
   - **全球代码**: 600519.SH
   - **交易场所**: 上海证券交易所
   - **参考价格**: 1800

### 步骤 2: 交易指标

1. 点击 **"进入下一阶段"**
2. 填写交易参数：
   - **名义承诺额**: 100000 (10万USDT)
   - **方向偏好**: 看涨 (Call) 或 看跌 (Put)
   - **期权费率目标**: 5%
   - **保证金储备比例**: 15%

### 步骤 3: 风险矩阵

1. 进入下一步
2. 设置：
   - **到期日期**: 选择 7-30 天后的日期
   - **流动性来源**: 自由卖方 / 席位持有者
   - **清算协议**: 连续涨跌停 或 波动激增
   - 如选"连续涨跌停"：设置连续天数 (2-5)
   - 如选"波动激增"：设置阈值百分比 (10-30%)

### 步骤 4: 协议访问

1. 进入下一步
2. 选择订单类型：
   - **开放网络**: 向所有卖方广播
   - **认证节点**: 仅验证过的对手方
3. 设置喂价规则

### 步骤 5: 最终确认

1. 核对所有参数
2. 检查 **钱包余额** 和 **协议授权额度**
3. 如授权不足，点击 **"授权协议"** 先进行 USDT 授权
4. 授权完成后，点击 **"提交并广播 RFQ"**
5. MetaMask 弹窗确认交易
6. 等待交易确认

### 预期结果

- [x] 显示 "信号广播成功" 成功页面
- [x] 自动跳转到交易大厅
- [x] 新创建的 RFQ 出现在列表中

## 6️⃣ 提交报价测试 (卖方流程)

### 前置条件

- 切换到 **卖方视角**
- 有活跃的 RFQ 可以报价

### 步骤

1. 在交易大厅找到一个 RFQ
2. 点击 **"提交报价"** 按钮
3. 进入报价页面，填写：
   - **报价费率 (%)**: 输入低于买方最高费率的值 (如 3.5%)
   - **保证金比例 (%)**: 输入高于最低要求 (如 15%)
   - **清算协议**: 选择 "已锁定" 或 "波动激增"
   - 如选"已锁定"：设置连续天数
   - 如选"波动激增"：设置阈值百分比
4. 查看 **财务预测** 区域显示的预期收益和锁仓要求
5. 点击 **"提交并广播提案"**
6. MetaMask 确认交易

### 预期结果

- [x] 显示 "报价已广播" 成功提示
- [x] 自动跳转回市场页面

## 7️⃣ 接受报价测试 (买方流程)

### 前置条件

- 切换到 **买方视角**
- 有已收到报价的 RFQ

### 步骤

1. 在交易大厅找到自己创建的 RFQ
2. 点击 **"查看报价"** 按钮
3. 弹窗显示所有收到的报价列表
4. 查看每个报价的：
   - 卖方地址
   - 报价费率

5. 选择最优报价，点击 **"接受"**
6. MetaMask 确认交易

### 预期结果

- [x] 显示成功提示
- [x] RFQ 状态变为 "MATCHED"
- [x] 订单出现在 "我的订单" 页面

## 9️⃣ 喂价面板测试

### 前置条件

- 已连接钱包
- 账户已注册为喂价员 (如未注册需先注册)

### 注册喂价员

1. 进入 **"喂价面板"**
2. 如显示 "注册预言机节点"，填写质押金额 (最低 100 USDT)
3. 点击 **"进入预言机网络"**
4. 确认交易

### 提交喂价

1. 查看 **信号队列** 中的待喂价任务
2. 点击任务的 **"提交信号"** 按钮
3. 弹窗中选择操作：
   - **确认参考价**: 确认当前价格正确
   - **修改价格**: 输入新价格并提交
   - **拒绝信号**: 选择拒绝原因并提交

- [x] 喂价成功后任务从队列消失

## 🐛 常见问题排查

### 问题 1: 页面白屏

```
检查浏览器控制台 (F12) 的错误信息
常见原因：
- BigInt 转换错误 → 刷新页面
- 钱包未连接 → 点击连接钱包
```

### 问题 2: 交易失败

```
检查：
1. 钱包 Gas 费是否充足
2. USDT 授权额度是否足够
3. 合约状态是否正常
```

### 问题 3: 数据不更新

```
尝试：
1. 点击 "重新加载" 按钮
2. 刷新页面 (F5)
3. 清除缓存后重试
```

### 问题 4: 语言切换不完整

```
部分文本未翻译通常是因为：
- 翻译键缺失
- 硬编码英文
请记录具体位置反馈修复
```
