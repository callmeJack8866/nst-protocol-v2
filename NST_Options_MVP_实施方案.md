## 1. 项目概述

### 1.1 项目目标

在 BSC（Binance Smart Chain）上构建一个完全链上闭环的场外期权撮合系统，具备以下核心能力：

- 买方或卖方任一方主导创建订单
- 对手方自由承接订单
- 独立喂价员生态进行人工喂价
- 链上自动清算 + 可配置仲裁倒计时

### 1.2 技术范围

| 项目     | 说明                              |
| -------- | --------------------------------- |
| 区块链   | Binance Smart Chain (BSC)         |
| 结算币种 | USDT / USDC（二选一，由订单定义） |
| 合约语言 | Solidity ^0.8.x                   |
| 前端框架 | React + Web3.js/Ethers.js         |
| 钱包支持 | MetaMask, WalletConnect           |

### 1.3 核心原则

1. **完全链上闭环**：所有关键资金托管于智能合约，无 off-chain 清算
2. **不穿仓原则**：买方最大盈利 = 卖方保证金
3. **协议零信用兜底**：NST 协议不承担任何对手方信用责任

### 2.2 各层职责详解

#### 2.2.1 治理层 (DAO Governance Layer)

| 合约                         | 职责                               | 权限边界                               |
| ---------------------------- | ---------------------------------- | -------------------------------------- |
| **GovernorNST**        | 提案、投票、DAO规则判定            | 不直接操作资金                         |
| **TimelockController** | 所有治理决议延迟执行（最少24小时） | 参数修改、合约升级、地址更换必须经过   |
| **AccessController**   | 统一角色权限管理                   | DAO / Operator / Guardian 三类角色分离 |

```solidity
// 角色定义
bytes32 constant DAO_ROLE = keccak256("DAO_ROLE");           // 治理决策
bytes32 constant OPERATOR_ROLE = keccak256("OPERATOR_ROLE"); // 日常运营
bytes32 constant GUARDIAN_ROLE = keccak256("GUARDIAN_ROLE"); // 紧急暂停
```

#### 2.2.2 控制层 (Core Control Layer)

| 合约                                   | 职责                       | 特性                                          |
| -------------------------------------- | -------------------------- | --------------------------------------------- |
| **Config / ParameterStore**      | 所有可调参数唯一存储       | 费率、分润比例、保证金规则、时间窗口、白名单  |
| **VaultManager**                 | 资金调度与状态执行中枢     | 只读Config，不拥有治理权，Owner必须是Timelock |
| **ProfitRouter / FeeController** | 统一收取手续费，分润到各池 | ProfitPool、DonationPool、节点收益等          |

```solidity
// 参数修改必须记录新旧值
event ParameterUpdated(
    string indexed paramName,
    uint256 oldValue,
    uint256 newValue,
    uint256 timestamp
);
```

#### 2.2.3 资产层 (Asset Vault Layer)

| 资金池                 | 用途                 |
| ---------------------- | -------------------- |
| **MarginPool**   | 卖方与买方保证金托管 |
| **ProfitPool**   | 协议利润池           |
| **TokenLPPool**  | 代币流动性与激励池   |
| **InteractPool** | 订单交互与手续费中转 |
| **DonationPool** | 捐赠与节点相关资金池 |

**安全规则**：所有Pool只信任VaultManager，不接受外部直接调用

#### 2.2.4 安全层 (Safety Layer)

| 功能                     | 说明                                  |
| ------------------------ | ------------------------------------- |
| **全局暂停**       | 一键暂停所有合约操作                  |
| **模块级暂停**     | 仅暂停特定模块（如仅暂停喂价）        |
| **Guardian权限**   | 只能pause，不能提取资金，不能修改参数 |
| **CircuitBreaker** | 异常检测自动熔断                      |

## 4. 智能合约设计

### 4.1 订单数据结构

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

struct Order {
    // 基础信息
    uint256 orderId;
    address buyer;
    address seller;

    // 标的信息（全部字符串，支持多语言）
    string underlyingName;      // 标的名称，如 "黄金", "AAPL"
    string underlyingCode;      // 标的代码，如 "XAU", "AAPL"
    string market;              // 市场，如 "CN", "US", "Crypto"
    string country;             // 国家，如 "CN", "US", "JP"
    string refPrice;            // 参考价格（文本，喂价员验证用）

    // 合约要素
    Direction direction;        // 看涨 Call / 看跌 Put
    uint256 notionalUSDT;       // 名义本金（已折算为USDT）
    uint256 strikePrice;        // 行权价（喂价后填入）
    uint256 expiryTimestamp;    // 到期时间戳

    // 费率与保证金
    uint256 premiumRate;        // 最终期权费率（基点，如 700 = 7%）
    uint256 premiumAmount;      // 期权费金额
    uint256 initialMargin;      // 卖方初始保证金（最低10%起）
    uint256 currentMargin;      // 卖方当前保证金
    uint256 minMarginRate;      // 最低保证金率（最低10%，可自定义更高）

    // 平仓规则
    LiquidationRule liquidationRule;
    uint8 consecutiveDays;      // 连续天数（1-10）
    uint8 dailyLimitPercent;    // 单日涨幅阈值（默认30%）
    uint8 exerciseDelay;        // T+X 行权延迟（1-5可选）

    // 卖方类型
    SellerType sellerType;
    address designatedSeller;   // 指定卖方地址（可选）

    // 仲裁与追保配置
    uint256 arbitrationWindow;  // 仲裁窗口（平仓结算24h/其他场景最长12h）
    uint256 marginCallDeadline; // 追保截止时间（2h/12h，根据标的类型）

    // 分红调整配置 ⭐ 新增
    bool dividendAdjustment;    // 分红调整：true=调整行权价，false=不调整

    // 状态与时间
    OrderStatus status;
    uint256 createdAt;
    uint256 matchedAt;
    uint256 settledAt;
    uint256 lastFeedPrice;      // 最后一次喂价价格（动态喂价无限制）
}

enum Direction {
    Call,   // 看涨
    Put     // 看跌
}

enum SellerType {
    FreeSeller,      // 自由卖方
    SeatSeller,      // 席位卖方
    DesignatedSeller // 指定卖方
}

enum LiquidationRule {
    NoLiquidation,           // 0: 无强平，到期结算
    ConsecutiveLimitUp,      // 1: 连续涨停板强平（触板/封板）
    ConsecutiveDailyGain     // 2: 连续单日涨幅≥X%强平
}

enum OrderStatus {
    RFQ_CREATED,              // 询价已创建
    QUOTING,                  // 报价中
    MATCHED,                  // 已匹配，待喂价
    WAITING_INITIAL_FEED,     // 待期初喂价
    LIVE,                     // 持仓中
    WAITING_FINAL_FEED,       // 待期末喂价
    PENDING_SETTLEMENT,       // 待结算
    ARBITRATION,              // 仲裁中
    SETTLED,                  // 已结算
    LIQUIDATED,               // 已强平
    CANCELLED                 // 已取消
}
```

### 4.4 喂价协议接口

```solidity
interface IFeedProtocol {
    /// @notice 喂价请求事件（供喂价员监听）
    event FeedRequested(
        uint256 indexed requestId,
        uint256 indexed orderId,
        string underlyingName,
        string underlyingCode,
        string market,
        string country,
        FeedType feedType,      // 期初/期中/期末/仲裁
        LiquidationRule liquidationRule, // 平仓规则同步给喂价员
        uint8 consecutiveDays,
        uint8 exerciseDelay,    // T+X行权条件
        uint256 timestamp
    );

    /// @notice 喂价提交事件
    event FeedSubmitted(
        uint256 indexed requestId,
        address indexed feeder,
        uint256 price,
        uint256 timestamp
    );

    /// @notice 喂价结果确认事件
    event FeedFinalized(
        uint256 indexed requestId,
        uint256 finalPrice,
        uint256 timestamp
    );

    /// @notice 喂价被拒绝事件（不符合T+X等条件）
    event FeedRejected(
        uint256 indexed requestId,
        address indexed feeder,
        string reason
    );

    enum FeedType {
        Initial,    // 期初喂价
        Dynamic,    // 动态保证金调整喂价（无次数限制）
        Final,      // 期末喂价
        Arbitration // 仲裁喂价（指定高级喂价员）
    }

    enum FeedTier {
        Tier_5_3,   // 5个喂价员，取3个有效
        Tier_7_5,   // 7个喂价员，取5个有效
        Tier_10_7   // 10个喂价员，取7个有效
    }

    /// @notice 创建喂价请求
    function createFeedRequest(
        uint256 orderId,
        FeedType feedType,
        FeedTier tier
    ) external payable returns (uint256 requestId);

    /// @notice 喂价员提交喂价
    function submitFeed(
        uint256 requestId,
        uint256 price
    ) external;

    /// @notice 喂价员拒绝喂价（不符合条件时）
    function rejectFeed(
        uint256 requestId,
        string calldata reason
```

### 9.2 喂价规则类型

| 规则类型               | 说明                   | 特点                             |
| ---------------------- | ---------------------- | -------------------------------- |
| **正常喂价**     | 按市场价格喂价         | 标准流程，始终可执行             |
| **跟量成交喂价** | 需要真实市场成交量支持 | 卖方提交建议价格，喂价员验证确认 |

### 9.2.1 跟量成交喂价数据结构 ⭐ 新增

```solidity
// 跟量成交喂价请求结构
struct VolumeBasedFeedRequest {
    uint256 requestId;
    uint256 orderId;
    address seller;              // 卖方地址
    uint256 suggestedPrice;      // 卖方建议成交价格
    string priceEvidence;        // 价格依据说明（可选）
    uint256 submittedAt;         // 提交时间
    bool isVerified;             // 喂价员是否验证通过
    address verifiedBy;          // 验证的喂价员地址
    uint256 finalPrice;          // 最终确认价格
    VolumeBasedFeedStatus status;
}

enum VolumeBasedFeedStatus {
    Pending,        // 待验证
    Approved,       // 验证通过，价格合理
    Rejected,       // 验证拒绝，价格不合理
    Modified        // 喂价员修正价格
}

// 卖方提交建议价格
event SuggestedPriceSubmitted(
    uint256 indexed requestId,
    uint256 indexed orderId,
    address indexed seller,
    uint256 suggestedPrice,
    uint256 timestamp
);

// 喂价员验证结果
event PriceVerified(
    uint256 indexed requestId,
    address indexed feeder,
    bool approved,
    uint256 finalPrice,         // 如果修正，为修正后价格
    string reason,              // 如果拒绝或修正，说明原因
    uint256 timestamp
);
```

```
├─────────────────────────────────────────────────────────────────────────────┤

│  │              → 原因：跟量成交没有价格，买卖不进去                    │   │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 11. 喂价生态系统

### 11.1 喂价协议独立性

喂价协议（FEED Protocol）与期权协议（NST Options）是**两套完全分离的协议**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    NST Options Protocol                                  │
│                    (期权撮合协议)                                        │
```
